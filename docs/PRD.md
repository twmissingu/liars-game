# Liars Game 产品需求文档 (PRD) - 优化重构版

## 1. 项目概述

### 1.1 项目背景
《Code Geass: Liar's Game》是一款基于《Code Geass》世界观的卡牌对战游戏，当前版本已实现基础玩法，需要进行优化重构以提升代码质量和用户体验。

### 1.2 项目目标
- 修复已知Bug，提升稳定性
- 优化代码结构，提高可维护性
- 完善测试覆盖，确保质量
- 补充文档，便于后续开发维护

---

## 2. 功能需求

### 2.1 核心玩法（已实现）
- **1v3人机对战**: 玩家同时对抗3个AI对手
- **骗子牌机制**: 每回合随机指定骗子牌，玩家可出真牌或撒谎
- **质疑系统**: 其他玩家可以质疑出牌，质疑成功触发Geass判定
- **角色技能**: 4个角色各有独特技能

### 2.2 已知Bug（待修复）

#### Bug 1: 圆桌牌背图片不显示 [P0]
- **现象**: 圆桌上的牌背图片没有正确渲染
- **影响**: 影响游戏视觉体验
- **预期**: 牌背应正确显示

#### Bug 2: 低分辨率页面无法滚动 [P0]
- **现象**: 在显示分辨率不高的浏览器页面，游戏页面不能上下滚动
- **影响**: 看不到玩家的头像信息
- **预期**: 页面应支持滚动，确保所有信息可见

#### Bug 3: 记录信息描述不明确 [P1]
- **现象**: 左侧记录信息缺少质疑成功或失败的结论，以及谁向谁发动了Geass
- **影响**: 玩家难以理解游戏进程
- **预期**: 记录应包含完整的质疑结果和Geass发动信息

#### Bug 4: 圆桌缺少出牌玩家信息 [P1]
- **现象**: 圆桌上只显示牌，不知道是谁出的牌
- **影响**: 玩家难以跟踪游戏状态
- **预期**: 应明确显示当前出牌玩家

#### Bug 5: AI出牌轮流逻辑错误 [P1]
- **现象**: 每次AI出牌后都轮到了玩家质疑或出牌，应该是4个玩家依次按顺序质疑或出牌
- **影响**: 游戏流程不符合设计
- **预期**: 严格按照回合顺序，4个玩家依次行动

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 3秒
- 游戏操作响应时间 < 100ms
- 支持主流浏览器（Chrome, Firefox, Safari, Edge）

### 3.2 兼容性要求
- 支持桌面端和移动端浏览器
- 支持不同分辨率（包括低分辨率设备）
- 响应式设计，适配各种屏幕尺寸

### 3.3 代码质量要求
- 代码注释覆盖率 > 80%
- 单元测试覆盖率 > 70%
- E2E测试覆盖核心流程
- 符合TypeScript类型规范

---

## 4. 技术架构

### 4.1 技术栈
- **框架**: React 18 + TypeScript
- **构建**: Vite
- **状态管理**: React Hooks + Context
- **样式**: CSS Modules / Styled Components
- **音效**: Howler.js

### 4.2 项目结构
```
src/
├── ai/                 # AI逻辑
├── characters/         # 角色系统
├── components/         # UI组件
├── core/              # 游戏核心引擎
├── data/              # 游戏数据
├── effects/           # 特效组件
├── store/             # 状态管理
├── styles/            # 全局样式
├── types/             # 类型定义
├── ui/                # 页面UI
└── utils/             # 工具函数
```
