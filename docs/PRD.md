## 1. 项目概述

### 1.1 项目背景
《Code Geass: Liar's Game》是一款基于《Code Geass》世界观的卡牌对战游戏，融合了经典动画角色与《骗子酒吧》的核心玩法。玩家扮演动画中的经典角色，与3个AI对手进行策略性的谎言对决，体验Geass能力的独特魅力。

### 1.2 项目目标
- **短期目标**: 修复已知Bug，提升游戏稳定性，优化用户体验
- **中期目标**: 完善角色技能平衡性，增强AI智能，扩展游戏内容
- **长期目标**: 支持多人联机对战，增加更多角色和玩法模式

### 1.3 目标用户画像

| 用户类型 | 特征 | 需求 |
|---------|------|------|
| Code Geass粉丝 | 熟悉动画剧情和角色 | 高度还原的角色形象和技能 |
| 卡牌游戏爱好者 | 喜欢策略和心理博弈 | 深度的策略玩法和平衡性 |
| 休闲游戏玩家 | 碎片化时间娱乐 | 简单易上手，单局时间短 |
| 策略游戏玩家 | 追求竞技和挑战 | AI难度分级，成就系统 |

## 2. 游戏核心玩法详细说明

### 2.1 游戏模式

#### 2.1.1 1v3人机对战（当前主模式）
- **玩家数量**: 1名真人玩家 + 3名AI对手
- **游戏目标**: 通过出牌和质疑，使所有对手HP归零
- **胜利条件**: 淘汰所有3个AI对手
- **失败条件**: 玩家HP归零

### 2.2 卡牌系统

#### 2.2.1 卡牌构成
- **牌组**: 使用标准扑克牌的Q、K、A三种点数
- **花色**: ♠黑桃、♥红桃、♣梅花、♦方块
- **总牌数**: 12张（3点数 × 4花色）
- **特殊牌**: 包含小丑牌(JOKER)，可当作任意点数使用

#### 2.2.2 卡牌属性
```typescript
interface Card {
  id: string;           // 唯一标识
  suit: CardSuit;       // 花色
  rank: CardRank;       // 点数 (Q/K/A)
  value: number;        // 数值权重
  isRevealed: boolean;  // 是否已揭示
  isJoker: boolean;     // 是否小丑牌
}
```

### 2.3 骗子牌机制

#### 2.3.1 机制说明
1. **每回合开始时**，系统随机指定一种点数为"骗子牌"
2. **出牌阶段**，玩家必须声称自己出的牌是骗子牌
3. **玩家可以选择**:
   - **出真牌**: 出的确实是骗子牌（诚实）
   - **撒谎**: 出的不是骗子牌，但声称是（欺骗）

#### 2.3.2 骗子牌轮换
- 每回合重新随机指定骗子牌
- 使用过的骗子牌本局不会重复出现（直到牌组重置）

### 2.4 回合流程

```
┌─────────────────────────────────────────────────────────────┐
│                        游戏回合流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 回合开始                                                │
│     ├── 随机指定骗子牌                                      │
│     └── 显示当前骗子牌                                      │
│                      ↓                                      │
│  2. 出牌阶段（当前玩家）                                     │
│     ├── 选择1-3张手牌（卡莲可出1-4张）                       │
│     ├── 系统自动声称是骗子牌                                │
│     └── 确认出牌                                            │
│                      ↓                                      │
│  3. 质疑阶段（其他3名玩家依次）                              │
│     ├── 玩家1：选择质疑/不质疑                              │
│     ├── 玩家2：选择质疑/不质疑                              │
│     └── 玩家3：选择质疑/不质疑                              │
│                      ↓                                      │
│  4. 结算阶段                                                │
│     ├── 如有质疑：进入Geass判定                             │

│                      ↓                                      │

│     ├── 命中：目标HP-1，触发搞笑动作                        │

│                      ↓                                      │
│  6. 回合结束                                                │
│     ├── 检查游戏结束条件                                    │
│     └── 切换到下一位玩家                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.5 质疑系统

#### 2.5.1 质疑规则
- **质疑时机**: 其他玩家出牌后，每位玩家依次决定是否质疑
- **质疑对象**: 质疑出牌玩家声称的牌是否真的是骗子牌
- **质疑结果**:
  - **质疑成功**: 出牌者在撒谎（出的牌不全是骗子牌）
  - **质疑失败**: 出牌者诚实（出的牌全是骗子牌）

#### 2.5.2 质疑后果
| 情况 | 结果 |
|------|------|
| 质疑成功 | 出牌者受Geass判定 |
| 质疑失败 | 质疑者受Geass判定 |
| 无人质疑 | 牌进入弃牌堆，回合继续 |

### 2.6 Geass判定系统

#### 2.6.1 基础机制
- **基础命中率**: 33.33% (1/3)
- **伤害值**: 每次命中造成1点HP伤害
- **最大HP**: 每位玩家3点HP
- **淘汰条件**: HP归零即被淘汰

#### 2.6.2 搞笑动作系统
Geass命中时，目标会执行随机搞笑动作：

| ID | 动作 | 描述 |
|----|------|------|
| 0 | 😵 | 突然跳起了奇怪的舞蹈 |
| 1 | 🙈 | 开始模仿猴子叫 |
| 2 | 🤪 | 不停地说"披萨" |
| 3 | 😂 | 无法控制地大笑30秒 |
| 4 | 🐔 | 学鸡打鸣 |
| 5 | 🎭 | 开始背诵中二台词 |
| 6 | 🍕 | 声称自己是披萨的化身 |
| 7 | 🦋 | 追逐不存在的蝴蝶 |

## 3. 角色技能详细设计

### 3.1 角色概览

| 角色 | 名称 | 主题色 | 难度 | 定位 |
|------|------|--------|------|------|
| 鲁鲁修 | 绝对命令 | #d4af37 (金) | ★★★☆☆ | 控场/策略 |
| C.C. | Code之力 | #22c55e (绿) | ★★☆☆☆ | 生存/续航 |
| 朱雀 | 枢木剑术 | #3b82f6 (蓝) | ★★★★☆ | 防御/反击 |
| 卡莲 | 红莲二式 | #dc2626 (红) | ★★★★★ | 进攻/爆发 |

### 3.2 鲁鲁修 - 绝对命令

#### 3.2.1 技能概述
- **技能名称**: 绝对命令 (Absolute Command)
- **技能类型**: 主动技能
- **使用次数**: 每局限用1次
- **冷却时间**: 无

#### 3.2.2 技能效果
强制改变当前骗子牌为任意指定点数（Q/K/A）。

#### 3.2.3 使用时机
- 自己回合开始时，出牌前使用
- 使用后立即生效，本回合骗子牌变为指定点数

#### 3.2.4 策略价值
- **高价值场景**: 
  - 手中有大量某一点数的牌时，可将骗子牌改为该点数
  - 配合队友（AI）手牌情况改变局势
- **反制手段**: 对手可通过观察鲁鲁修手牌数量变化判断其是否使用技能

#### 3.2.5 技能台词
> "我以鲁鲁修·vi·布里塔尼亚之名下令..."

### 3.3 C.C. - Code之力

#### 3.3.1 技能概述
- **技能名称**: Code之力 (Power of Code)
- **技能类型**: 被动技能
- **使用次数**: 每局限1次
- **触发条件**: 首次HP归零时

#### 3.3.2 技能效果
首次濒死时，有50%概率复活并恢复1点HP。

#### 3.3.3 机制细节
- 复活后HP为1点
- 复活后技能失效，再次归零即被淘汰
- 复活时触发特殊动画效果

#### 3.3.4 策略价值
- **高价值场景**: 
  - 残血时更敢于冒险出牌
  - 对手不敢轻易质疑C.C.的出牌
- **反制手段**: 对手可在C.C.复活后集中火力快速淘汰

#### 3.3.5 技能台词
> "我是不老不死的魔女..."

### 3.4 朱雀 - 枢木剑术

#### 3.4.1 技能概述
- **技能名称**: 枢木剑术 (Kururugi Swordsmanship)
- **技能类型**: 被动技能
- **使用次数**: 无限
- **触发条件**: 受到Geass判定时

#### 3.4.2 技能效果
- **闪避**: 15%概率完全闪避Geass
- **反击**: 闪避成功时，25%概率反击，使攻击者受到Geass判定

#### 3.4.3 机制细节
- 闪避和反击独立计算
- 反击命中率为基础33%
- 反击可触发搞笑动作

#### 3.4.4 策略价值
- **高价值场景**: 
  - 残血时生存率显著提升
  - 对手质疑朱雀时更谨慎
- **反制手段**: 多人轮流质疑，增加朱雀受击次数

#### 3.4.5 技能台词
> "我要从内部改变世界！"

### 3.5 卡莲 - 红莲二式

#### 3.5.1 技能概述
- **技能名称**: 红莲二式 (Guren Mk-II)
- **技能类型**: 主动/被动混合
- **使用次数**: 每回合可用
- **特殊规则**: 每回合可出1-4张牌（其他角色最多3张）

#### 3.5.2 技能效果
- **多牌出牌**: 每回合可出1-4张牌
- **Geass强化**: 出2张以上牌且质疑失败时，Geass命中率 = 20% × 出牌张数

#### 3.5.3 机制细节
- 出2张牌：命中率40%
- 出3张牌：命中率60%
- 出4张牌：命中率80%
- 强化效果仅对质疑失败的质疑者生效

#### 3.5.4 策略价值
- **高价值场景**: 
  - 快速出完手牌获得优势
  - 多张牌出牌时威慑对手不敢质疑
- **风险**: 出牌越多，被质疑成功的损失越大

#### 3.5.5 技能台词
> "黑色骑士团，突入！"

### 3.6 角色平衡性分析

| 角色 | 优势 | 劣势 | 适合玩家 |
|------|------|------|----------|
| 鲁鲁修 | 控场能力强，可改变局势 | 技能限用1次，容错率低 | 策略型玩家 |
| C.C. | 生存能力强，容错率高 | 无主动进攻能力 | 新手/稳健型 |
| 朱雀 | 防御反击，残血强势 | 依赖概率，不稳定 | 风险偏好型 |
| 卡莲 | 爆发力强，快速结束战斗 | 高风险高回报 | 高手/激进型 |

## 6. 已知Bug清单

### 6.1 P0级Bug（严重）

#### Bug 1: 圆桌牌背图片不显示
- **现象**: 圆桌上的牌背图片没有正确渲染
- **影响**: 影响游戏视觉体验，玩家无法看到桌面上的牌
- **预期**: 牌背应正确显示统一图案
- **可能原因**: 图片路径错误或资源加载失败

#### Bug 2: 低分辨率页面无法滚动
- **现象**: 在显示分辨率不高的浏览器页面，游戏页面不能上下滚动
- **影响**: 看不到玩家的头像信息，无法完整游戏
- **预期**: 页面应支持滚动，确保所有信息可见
- **可能原因**: CSS overflow设置或固定高度问题

### 6.2 P1级Bug（重要）

#### Bug 3: 记录信息描述不明确
- **现象**: 左侧记录信息缺少质疑成功或失败的结论，以及谁向谁发动了Geass
- **影响**: 玩家难以理解游戏进程，降低游戏体验
- **预期**: 记录应包含完整的质疑结果和Geass发动信息
- **示例**: "C.C. 质疑 鲁鲁修！质疑成功！C.C. 向 鲁鲁修 发动Geass！命中！"

#### Bug 4: 圆桌缺少出牌玩家信息
- **现象**: 圆桌上只显示牌，不知道是谁出的牌
- **影响**: 玩家难以跟踪游戏状态，容易混淆
- **预期**: 应明确显示当前出牌玩家名称或头像

#### Bug 5: AI出牌轮流逻辑错误
- **现象**: 每次AI出牌后都轮到了玩家质疑或出牌，应该是4个玩家依次按顺序质疑或出牌
- **影响**: 游戏流程不符合设计，影响策略性
- **预期**: 严格按照回合顺序，4个玩家依次行动
- **说明**: 当前逻辑可能跳过了某些AI的行动回合
