## 1. 项目概述

### 1.1 项目背景
《Code Geass: Liar's Game》是一款基于《Code Geass》世界观的卡牌对战游戏，融合了经典动画角色与《骗子酒吧》的核心玩法。玩家扮演动画中的经典角色，与3个AI对手进行策略性的谎言对决，体验Geass能力的独特魅力。

### 1.2 项目目标
- **短期目标**: 修复已知Bug，提升游戏稳定性，优化用户体验
- **中期目标**: 完善角色技能平衡性，增强AI智能，扩展游戏内容
- **长期目标**: 支持多人联机对战，增加更多角色和玩法模式

### 1.3 目标用户画像

| 用户类型 | 特征 | 需求 |
|---------|------|------|
| Code Geass粉丝 | 熟悉动画剧情和角色 | 高度还原的角色形象和技能 |
| 卡牌游戏爱好者 | 喜欢策略和心理博弈 | 深度的策略玩法和平衡性 |
| 休闲游戏玩家 | 碎片化时间娱乐 | 简单易上手，单局时间短 |
| 策略游戏玩家 | 追求竞技和挑战 | AI难度分级，成就系统 |

## 2. 游戏核心玩法详细说明

### 2.1 游戏模式

#### 2.1.1 1v3人机对战（当前主模式）
- **玩家数量**: 1名真人玩家 + 3名AI对手
- **游戏目标**: 通过出牌和质疑，使所有对手HP归零
- **胜利条件**: 淘汰所有3个AI对手
- **失败条件**: 玩家HP归零

### 2.2 卡牌系统

#### 2.2.1 卡牌构成
- **牌组**: 使用标准扑克牌的Q、K、A三种点数
- **花色**: ♠黑桃、♥红桃、♣梅花、♦方块
- **总牌数**: 12张（3点数 × 4花色）
- **特殊牌**: 包含小丑牌(JOKER)，可当作任意点数使用

#### 2.2.2 卡牌属性
```typescript
interface Card {
  id: string;           // 唯一标识
  suit: CardSuit;       // 花色
  rank: CardRank;       // 点数 (Q/K/A)
  value: number;        // 数值权重
  isRevealed: boolean;  // 是否已揭示
  isJoker: boolean;     // 是否小丑牌
}
```

### 2.3 骗子牌机制

#### 2.3.1 机制说明（Liar's Bar核心规则）
1. **每回合开始时**，系统从Q、K、A中随机指定一种点数为"骗子牌"（Table Card）
2. **出牌阶段**，玩家出牌并声称这些牌都是骗子牌
3. **玩家出牌选择**:
   - **出真牌**: 出的牌确实都是骗子牌（诚实）
   - **撒谎**: 出的牌中包含非骗子牌，但声称全是骗子牌（欺骗）
   - **使用小丑牌**: 小丑牌（JOKER）是万能牌，可当任意点数使用

#### 2.3.2 骗子牌轮换规则
- 每回合开始时重新随机指定骗子牌（Q/K/A之一）
- 骗子牌可以在不同回合重复出现
- 当牌组耗尽时，重新洗牌并继续游戏

#### 2.3.3 小丑牌规则（JOKER）
- **数量**: 牌组中包含2张小丑牌
- **功能**: 小丑牌可当任意点数使用，视为骗子牌
- **判定**: 质疑时，小丑牌自动匹配当前声称的点数，不会被视为撒谎
- **策略价值**: 
  - 可以安全地配合真牌一起出，增加出牌数量
  - 提高质疑成功率时的威慑力（因为小丑牌不会暴露谎言）

#### 2.3.4 出牌数量规则
- **标准出牌**: 每回合可出1-3张牌（卡莲角色可出1-4张）
- **混合出牌**: 可以同时出骗子牌和非骗子牌（撒谎）
- **全小丑牌**: 可以只出小丑牌，此时无论声称什么点数都不算撒谎

#### 2.4.2 出牌阶段详细规则

**出牌限制**:
- 每回合必须至少出1张牌，最多3张（卡莲角色可出4张）
- 出牌时系统自动声称这些牌都是当前骗子牌
- 玩家可以选择诚实出牌（出的都是骗子牌或小丑）或撒谎（混入非骗子牌）

**出牌策略**:
- **保守策略**: 只出骗子牌，安全但进度慢
- **激进策略**: 混入非骗子牌快速消耗手牌，但风险高
- **混合策略**: 出1张真牌+小丑牌，相对安全且有威慑力

#### 2.4.3 质疑阶段详细规则

**质疑顺序**:
1. 严格按照出牌者的下家开始，顺时针方向依次进行
2. 每位玩家在自己的质疑回合可以选择质疑或不质疑
3. 一旦有人质疑，立即翻牌结算，不再询问后续玩家
4. 如果所有人都选择不质疑，则质疑阶段结束

**质疑决策**:
- **质疑时机**: 怀疑对方在撒谎时发起质疑
- **质疑风险**: 质疑失败自己将受Geass判定
- **不质疑**: 继续游戏，轮到下一位玩家行动

**示例流程**:
```
玩家出牌 → AI1（下家）选择质疑 → 翻牌结算 → Geass判定 → 回合结束
玩家出牌 → AI1 不质疑 → AI2 选择质疑 → 翻牌结算 → Geass判定 → 回合结束
玩家出牌 → AI1 不质疑 → AI2 不质疑 → AI3 不质疑 → 无人质疑 → 回合继续
```

### 2.5 质疑系统

#### 2.5.1 质疑规则（Liar's Bar标准规则）

**质疑时机**:
- 其他玩家出牌后，进入质疑阶段
- 从出牌者的下家开始，按顺时针顺序依次询问
- 每位玩家在自己的质疑回合可以选择质疑或不质疑

**质疑对象**:
- 质疑出牌玩家声称的牌是否都是骗子牌
- 即：质疑"出的牌中是否包含非骗子牌（且非小丑）"

**质疑结果判定**:

| 情况 | 判定标准 | 结果 |
|------|----------|------|
| **质疑成功** | 出牌者出的牌中包含非骗子牌（小丑牌除外） | 出牌者在撒谎，受Geass判定 |
| **质疑失败** | 出牌者出的牌全是骗子牌或小丑牌 | 出牌者诚实，质疑者受Geass判定 |

**判定示例**:
- 骗子牌是K，玩家声称出的是K，实际出的牌是[K♠, K♥] → 诚实，质疑失败
- 骗子牌是K，玩家声称出的是K，实际出的牌是[K♠, Q♥] → 撒谎，质疑成功
- 骗子牌是K，玩家声称出的是K，实际出的牌是[K♠, JOKER] → 诚实（小丑当K用），质疑失败

**质疑顺序**:
1. 出牌者下家（顺时针第一位）首先决定是否质疑
2. 如果选择质疑 → 立即翻牌结算，结束质疑阶段
3. 如果选择不质疑 → 继续询问下一位玩家
4. 重复直到有人质疑或所有玩家都选择不质疑

**质疑决策策略**:

| 策略类型 | 适用场景 | 风险 |
|----------|----------|------|
| **积极质疑** | 对方出牌数量多、对方手牌少、自己HP充足 | 质疑失败自己受罚 |
| **保守不质疑** | 对方出牌数量少、自己HP低、不确定对方是否撒谎 | 对方可能快速出完手牌获胜 |
| **观察等待** | 游戏早期、信息不足时 | 可能错过质疑机会 |

#### 2.5.3 质疑后果

| 情况 | 结果 | 受罚者 |
|------|------|--------|
| **质疑成功** | 出牌者在撒谎，翻牌后暴露谎言 | 出牌者受Geass判定 |
| **质疑失败** | 出牌者诚实，质疑者判断错误 | 质疑者受Geass判定 |
| **无人质疑** | 所有玩家选择相信，牌进入弃牌堆 | 无人受罚，回合继续 |

#### 2.5.4 特殊规则

**小丑牌保护**:
- 小丑牌在质疑判定时自动匹配声称的点数
- 出小丑牌不会被视为撒谎
- 可以安全地配合真牌一起出

**连续质疑限制**:
- 一轮质疑阶段中，一旦有人质疑，立即结算，不再询问其他玩家
- 这意味着只有最先质疑的玩家承担质疑失败的风险/获得质疑成功的收益

### 2.6 Geass判定系统

#### 2.6.1 基础机制（Liar's Bar惩罚机制）

**Geass判定**是Liar's Bar游戏中的核心惩罚机制，当质疑成功或失败时触发。

| 属性 | 数值 | 说明 |
|------|------|------|
| **基础命中率** | 33.33% (1/3) | 每次Geass判定有1/3概率命中 |
| **伤害值** | 1点HP | 每次命中造成1点伤害 |
| **最大HP** | 3点（朱雀4点） | 每位玩家的生命值上限 |
| **淘汰条件** | HP归零 | HP降为0时玩家被淘汰出局 |

**判定流程**:
1. 确定受罚者（质疑成功→出牌者，质疑失败→质疑者）
2. 计算实际命中率（基础33.33% ± 技能修正）
3. 执行随机判定（Math.random() < 命中率）
4. 命中：HP-1，触发搞笑动作；未命中：无事发生

#### 2.6.2 角色技能对Geass的影响

| 角色 | 技能名称 | 技能效果 | 对Geass的影响 |
|------|----------|----------|---------------|
| **C.C.** | Code之力 | 被动技能，首次濒死时50%概率复活 | 免疫本次Geass伤害，恢复1HP |
| **朱雀** | 枢木剑术 | 被动技能，15%闪避 + 25%反击 | 闪避：完全免疫；反击：攻击者受Geass |
| **卡莲** | 红莲二式 | 主动技能，出2张+被质疑失败时 | Geass命中率 = 20% × 出牌张数 |
| **鲁鲁修** | 绝对命令 | 主动技能，改变骗子牌 | 间接影响，不直接改变Geass判定 |

**技能详细说明**:

**C.C. - Code之力（复活）**:
- 触发条件：首次受到致命伤害（HP会从1变为0）
- 效果：50%概率免疫本次伤害并恢复1HP
- 限制：每局限1次，复活后技能失效

**朱雀 - 枢木剑术（闪避+反击）**:
- 闪避：受到Geass时有15%概率完全闪避
- 反击：闪避成功后有25%概率反击，使攻击者受Geass判定
- 反击命中率：基础33.33%

**卡莲 - 红莲二式（命中率加成）**:
- 触发条件：出2张以上牌且质疑失败（对方质疑卡莲失败）
- 效果：Geass命中率 = 20% × 出牌张数
- 示例：出2张→40%，出3张→60%，出4张→80%

#### 2.6.3 搞笑动作系统

Geass命中后，受罚者需要执行随机搞笑动作，增加游戏趣味性。

| ID | 动作 | 描述 | 音效 |
|----|------|------|------|
| 0 | 😵 | 突然跳起了奇怪的舞蹈 | sfx-funny-dance |
| 1 | 🙈 | 开始模仿猴子叫 | sfx-funny-monkey |
| 2 | 🤪 | 不停地说"披萨" | sfx-funny-pizza |
| 3 | 😂 | 无法控制地大笑30秒 | sfx-funny-laugh |
| 4 | 🐔 | 学鸡打鸣 | sfx-funny-chicken |
| 5 | 🎭 | 开始背诵中二台词 | sfx-funny-chunibyo |
| 6 | 🍕 | 声称自己是披萨的化身 | sfx-funny-pizza |
| 7 | 🦋 | 追逐不存在的蝴蝶 | sfx-funny-butterfly |

**搞笑动作触发**:
- Geass命中时随机选择
- 播放对应动画和音效
- 在战斗日志中显示

#### 2.6.4 游戏结束判定

**淘汰条件**:
- 玩家HP归零时立即被淘汰
- 被淘汰玩家不再参与后续回合
- 手牌清空，不再被轮到行动

**胜利条件**:
- 最后存活的玩家获胜
- 1v3模式中，玩家淘汰所有3个AI → 玩家胜利
- 1v3模式中，玩家HP归零 → AI胜利

**游戏结束流程**:
1. 检查是否有玩家HP归零
2. 淘汰HP归零的玩家
3. 检查存活玩家数量
4. 只剩1名玩家 → 游戏结束，显示胜利/失败画面
5. 多于1名玩家 → 继续游戏，轮到下一个存活玩家

### 3.5 卡莲 - 红莲二式

#### 3.5.1 技能概述
- **技能名称**: 红莲二式 (Guren Mk-II)
- **技能类型**: 主动/被动混合
- **使用次数**: 每回合可用
- **特殊规则**: 每回合可出1-4张牌（其他角色最多3张）

#### 3.5.2 技能效果
- **多牌出牌**: 每回合可出1-4张牌
- **Geass强化**: 出2张以上牌且质疑失败时，Geass命中率 = 20% × 出牌张数

#### 3.5.3 机制细节
- 出2张牌：命中率40%
- 出3张牌：命中率60%
- 出4张牌：命中率80%
- 强化效果仅对质疑失败的质疑者生效

#### 3.5.4 策略价值
- **高价值场景**: 
  - 快速出完手牌获得优势
  - 多张牌出牌时威慑对手不敢质疑
- **风险**: 出牌越多，被质疑成功的损失越大

#### 3.5.5 技能台词
> "黑色骑士团，突入！"

## 6. 已知Bug清单

### 6.1 P0级Bug（严重）

#### Bug 1: 圆桌牌背图片不显示
- **现象**: 圆桌上的牌背图片没有正确渲染
- **影响**: 影响游戏视觉体验，玩家无法看到桌面上的牌
- **预期**: 牌背应正确显示统一图案
- **可能原因**: 图片路径错误或资源加载失败

#### Bug 2: 低分辨率页面无法滚动
- **现象**: 在显示分辨率不高的浏览器页面，游戏页面不能上下滚动
- **影响**: 看不到玩家的头像信息，无法完整游戏
- **预期**: 页面应支持滚动，确保所有信息可见
- **可能原因**: CSS overflow设置或固定高度问题

### 6.2 P1级Bug（重要）

#### Bug 3: 记录信息描述不明确
- **现象**: 左侧记录信息缺少质疑成功或失败的结论，以及谁向谁发动了Geass
- **影响**: 玩家难以理解游戏进程，降低游戏体验
- **预期**: 记录应包含完整的质疑结果和Geass发动信息
- **示例**: "C.C. 质疑 鲁鲁修！质疑成功！C.C. 向 鲁鲁修 发动Geass！命中！"

#### Bug 4: 圆桌缺少出牌玩家信息
- **现象**: 圆桌上只显示牌，不知道是谁出的牌
- **影响**: 玩家难以跟踪游戏状态，容易混淆
- **预期**: 应明确显示当前出牌玩家名称或头像

#### Bug 5: AI出牌轮流逻辑错误
- **现象**: 每次AI出牌后都轮到了玩家质疑或出牌，应该是4个玩家依次按顺序质疑或出牌
- **影响**: 游戏流程不符合设计，影响策略性
- **预期**: 严格按照回合顺序，4个玩家依次行动
- **说明**: 当前逻辑可能跳过了某些AI的行动回合
